# State Machines

## Order Status (`order_status`)

```
                    ┌─────────┐
                    │   NEW   │ ◄── створення замовлення
                    └────┬────┘
                         │
                    ┌────▼──────┐
               ┌────│ CONFIRMED │
               │    └────┬──────┘
               │         │
               │    ┌────▼───────┐
               │    │ PROCESSING │
               │    └────┬───────┘
               │         │
               │    ┌────▼────┐
               ├────│ SHIPPED │──────────┐
               │    └────┬────┘          │
               │         │              │
               │    ┌────▼──────┐  ┌────▼─────┐
               │    │ DELIVERED │  │ RETURNED  │
               │    └───────────┘  └──────────┘
               │
          ┌────▼──────┐
          │ CANCELLED │
          └───────────┘
```

### Дозволені переходи

| З | В | Коментар |
|---|---|----------|
| `NEW` | `CONFIRMED` | Адмін підтвердив замовлення |
| `NEW` | `CANCELLED` | Адмін скасував (немає товару, помилка, тощо) |
| `CONFIRMED` | `PROCESSING` | Замовлення в обробці (збірка) |
| `CONFIRMED` | `CANCELLED` | Скасування до відправки |
| `PROCESSING` | `SHIPPED` | Передано перевізнику (ТТН виписана) |
| `PROCESSING` | `CANCELLED` | Скасування до відправки |
| `SHIPPED` | `DELIVERED` | Клієнт отримав |
| `SHIPPED` | `RETURNED` | Повернення після відправки |

**Note:** Поточна реалізація не валідує переходи — адмін може встановити будь-який статус через `PATCH /orders/:id/status`. Валідація переходів — кандидат на майбутню реалізацію.

## Payment Status (`payment_status`)

```
  ┌─────────┐
  │ PENDING │ ◄── створення замовлення (завжди)
  └────┬────┘
       │
       ├──────────┐
       │          │
  ┌────▼──┐  ┌───▼────┐
  │ PAID  │  │ FAILED │
  └───┬───┘  └────────┘
      │
  ┌───▼──────┐
  │ REFUNDED │
  └──────────┘
```

### Дозволені переходи

| З | В | Коментар |
|---|---|----------|
| `PENDING` | `PAID` | Оплата підтверджена (адмін вручну або інтеграція) |
| `PENDING` | `FAILED` | Оплата не пройшла |
| `PAID` | `REFUNDED` | Повернення коштів |

**Note:** Оплата наразі підтверджується адміном вручну (IBAN). Автоматичне підтвердження — при реалізації LiqPay/MonoPay.

## Product Variant Status (`status`)

```
  ┌───────┐
  │ DRAFT │ ◄── створення (якщо вказано)
  └───┬───┘
      │
  ┌───▼────┐
  │ ACTIVE │ ◄── за замовчуванням при створенні
  └───┬────┘
      │
  ┌───▼──────┐
  │ ARCHIVED │
  └──────────┘
```

| Статус | В каталозі | В замовленні |
|--------|-----------|-------------|
| `draft` | Не показується | Не можна додати |
| `active` | Показується | Можна додати |
| `archived` | Не показується | Існуючі замовлення зберігаються |

Фільтр каталогу: `status = "active"` (hardcoded в агрегації).

## Discount Coupon Lifecycle

```
  Створення (адмін)
       │
       ▼
  ┌──────────────┐
  │  is_active   │ ◄── true за замовчуванням
  │  valid_until │ ◄── дата закінчення
  └──────┬───────┘
         │
    Валідація при checkout:
         │
         ├── code не знайдено      → { valid: false, reason: "NOT_FOUND" }
         ├── is_active = false     → { valid: false, reason: "INACTIVE" }
         ├── valid_until < now     → { valid: false, reason: "EXPIRED" }
         └── все ок                → { valid: true, coupon: {...} }

    При створенні замовлення:
         │
         └── Snapshot в order.applied_discount:
             { coupon_id, code, discount_percent, discount_amount }
```

**Note:** Купон не має лічильника використань — один код можна використати необмежену кількість разів до закінчення терміну дії або деактивації адміном.
